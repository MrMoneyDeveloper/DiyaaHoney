Welcome to DiyaaHoney

This project demonstrates a honeypot‑based intrusion detection system and a management dashboard for incident response. It captures connection attempts on a decoy service, analyzes attack patterns, and stores events in a relational database. The system also exposes a web interface for monitoring activity, exploring data and performing administrative tasks.

Business Processes

  1. Security Incident Detection & Response – The honeypot listens on a predefined port and records every inbound connection. An intrusion detector analyzes the log and triggers alerts when suspicious behaviour (e.g. repeated brute‑force attempts) is detected. Alerts are persisted to the database and delivered via e‑mail, MQTT and optional on‑device indicators.
  2. System Administration & Reporting – Administrators manage user accounts, review accumulated connection data and alerts, run queries and purge outdated records. The dashboard supports searching and filtering, while the database GUI provides a safe read‑only environment for exploring tables via SQL.

Use Cases

  • Record Connection: Each time a client connects to the honeypot, the IP, port and timestamp are stored in the connections table.
  • Auto‑Alert on Brute‑Force: The intrusion detector counts failed attempts per address; once a threshold is crossed it inserts an entry in alerts and sends notifications.
  • User Login: Users authenticate via the web interface; sessions are managed securely with Flask‑Login and hashed passwords.
  • Search Connections: Analysts can filter the dashboard by IP or date range to investigate suspicious patterns.
  • Purge Old Data: Administrators can remove obsolete connection records older than a specified retention period; associated alerts are deleted automatically via cascading constraints.
